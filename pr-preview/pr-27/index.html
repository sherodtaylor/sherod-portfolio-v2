Selected projects

agent-smith · Framework docs

Autonomous AI engineering teammates running in a home-lab Kubernetes cluster. Matrix is the input. Tag a bot, it opens a PR; tag another, it reviews. They never hold real credentials — those stay behind an egress proxy. Public Helm chart.

  • Three agents, one image: DevBot (code), InfraBot (k3s/Flux), PMBot (product/PRDs) run as separate StatefulSets from the same parametric image. Per-agent persona is a directory under agents/<name>/
  • Matrix-driven PR workflow: messages in #dev / #infra are real Claude Code prompts. Author opens a PR; the other agent reviews and posts inline findings. A Stop-hook re-wakes the author on unaddressed comments — iteration without a human in the loop
  • Egress credential firewall (iron-proxy): agents never hold real GitHub or Anthropic tokens. iron-proxy MITMs egress and swaps placeholders for real creds per-host against an allowlist. A compromised pod leaks nothing useful
  • MCP for the cluster's own observability: VictoriaMetrics, VictoriaLogs, NATS, and the Matrix channel are all MCP servers. 'Check the api-latency dashboard' is one prompt
  • Public OSS: image at ghcr.io/sherodtaylor/agent-smith, Helm chart at oci://ghcr.io/sherodtaylor/charts/agent-smith

Home Lab Cluster

Self-hosted k3s cluster running every app my family depends on. Flux reconciles from one git repo, ExternalSecrets pulls creds from Infisical, VictoriaMetrics + VictoriaLogs catch drift. Family-grade, GitOps-managed.

  • Media + photos + files: Jellyfin, Plex, Immich (face-recognition over years of family photos), Nextcloud, Audiobookshelf — all running on the cluster, backed by TrueNAS NFS
  • Home Automation: Home Assistant — lights, thermostats, schedules, sensors — on the same cluster, not a separate appliance
  • Observability built-in: every pod's metrics and logs land in VictoriaMetrics / VictoriaLogs. Grafana + Alertmanager catch regressions before they become outages
  • GitOps all the way down: Flux reconciles every Kustomization and HelmRelease. cert-manager + kubernetes-replicator handle TLS. The cluster rebuilds from a single git push
